--------------------------------------------------------------------------------
  OVERVIEW
--------------------------------------------------------------------------------

Last Stand Arena is a fully playable first/third-person 3D battle royale shooter
built entirely from scratch in Python using raw OpenGL fixed-function pipeline
calls. No game engine, no physics library, no scene graph — every triangle,
every lighting calculation, and every collision is hand-written.

The project started as a university class assignment and grew into a complete
game with a weapon system, enemy AI, power-ups, UI overlays, and an FPS limiter.

--------------------------------------------------------------------------------
  TECHNICAL SKILLS DEMONSTRATED
--------------------------------------------------------------------------------

Graphics & Rendering
  - OpenGL fixed-function pipeline (glBegin/glEnd, display lists, fog, blending)
  - 3D scene construction: cylinders, spheres, cones, tori via GLUT primitives
  - Custom first-person weapon models (distinct pistol / AR / shotgun meshes)
  - 3D power-up shapes: cross, diamond, spinning star, dome
  - Muzzle flash effect using additive blending (GL_ONE)
  - HUD / 2D overlay rendering with orthographic projection
  - Floating damage numbers and hit markers
  - Minimap with enemy blips

Camera System
  - Dual camera modes: first-person (FPS) and third-person orbital
  - Smooth camera lag with lerp interpolation
  - Mouse-based yaw and pitch with gimbal-lock avoidance
  - Arrow-key zoom in third-person mode

Game Architecture
  - Single-file Python game engine (~2700 lines)
  - Entity-component style dictionaries for enemies, bullets, power-ups
  - Wave-based enemy spawning with difficulty scaling
  - Collision detection: AABB sphere vs sphere, bullet-enemy overlap
  - Zone-aware hit detection: headshots, body shots, leg shots
  - Accurate per-weapon damage/accuracy/fire-rate/reload model

Weapon System
  - Three weapons: Pistol, Assault Rifle, Shotgun
  - Each weapon has: damage, fire rate, pellet count, ammo pool, reload time
  - Accuracy tracks total shots vs hits at pellet granularity (shotgun fix)
  - Headshot detection uses height zones + 25% probability gate

Enemy AI
  - Line-of-sight patrol → chase → attack state machine
  - Health bars rendered as 3D billboard quads
  - Multiple enemy types with different speed/health/armor

Power-Ups
  - Four types: Health, Ammo, Speed boost, Shield
  - Distinct 3D shapes per type; animated rotation
  - Collected on proximity

UI / UX
  - Bitmap font rendering for HUD text (glutBitmapCharacter)
  - Kill feed, crosshair, ammo counter, health/shield bars
  - Name entry screen, main menu, game over / victory screens
  - FPS limiter (GLUT idle callback with sleep)
  - Pause / resume with overlay

Input Handling
  - GLUT keyboard callbacks (normal + special keys)
  - Sprint modifier (Shift), weapon switch (1/2/3)
  - Passive mouse motion for camera yaw/pitch

--------------------------------------------------------------------------------
  DESIGN DECISIONS
--------------------------------------------------------------------------------

1. Single-file architecture
   Keeps the project portable and easy to submit / share.
   All rendering, logic, input, and AI in one file following a clear class
   hierarchy: Engine → Scene → Entities → Renderer → HUD.

2. Fixed-function OpenGL (not shaders)
   Deliberately avoids GLSL to demonstrate understanding of the underlying
   transformation pipeline (modelview stack, normal vectors, material properties,
   lighting equations) without shader abstraction.

3. Python + GLUT over Pygame/Panda3D
   Maximises control over the render loop and demonstrates low-level API
   usage relevant to graphics programming coursework.

4. Per-pellet accuracy tracking
   Shotgun fires multiple pellets per shot; naive "+1 per shot" inflated
   accuracy to 100%. Fixed by accumulating pellet count in total_shots_fired.

5. Layered headshot logic
   Height-zone detection first (only the top ~25% of enemy height qualifies),
   then a 25% probability gate — prevents guaranteed headshots while still
   rewarding aimed shots.

--------------------------------------------------------------------------------
  GAMEPLAY SUMMARY
--------------------------------------------------------------------------------

  Gamemode     : Wave-based survival (battle royale style)
  Perspective  : First-person (default) or third-person (toggle with F)
  Weapons      : Pistol, Assault Rifle, Shotgun (switch with 1/2/3)
  Enemies      : Multiple types, increasing waves
  Power-ups    : Health, Ammo, Speed, Shield
  Win Condition: Survive all waves
  Lose Condition: Health reaches zero
